# 产品大师 Pro：7 个 Skill，覆盖产品从 0 到 1 的全流程

做产品最难的事，不是写代码，而是想清楚。

---

## 为什么做这个？

入行的时候，Pony 的"马氏三问"是新人必修课——你擅长吗？不做用户损失什么？能保持多大优势？三个问题，简单到朴素，但真正能答好的需求不多。后来翻烂了 Steve Krug 的 *Don't Make Me Think*，学会了一件事：用户没有耐心，你每多一步操作就多丢一批人。再后来，赶上张小龙那场旷古绝今的 8 小时分享，从产品哲学讲到人性，讲到人的贪嗔痴，讲到"好的产品用完即走"。这些东西在脑子里扎了根，变成了日后做每一个产品决策的底层操作系统。

后来去了字节做产品，见识了另一套打法——数据驱动、快速迭代、AB测试跑通一切。再后来 GPT 出来，在和 AI 深度协作共同推进产品的过程中，发现了一个结合点：这些年积累下来的产品方法论，如果变成结构化的 Prompt，AI 就能像一个经验丰富的产品搭档一样，帮你把该想的问题全部想一遍。

于是有了这组 Skill。

---

## 这组 Skill 解决什么问题？

产品经理的日常，大概是这样的：

**周一**，老板说"我想做一个XX平台"。你点头说好，但心里没底——这个"平台"到底要解决什么问题？给谁用？

**周二**，你做了份用户调研。100个人说"挺好的，我会用"。但你知道，问卷上说"会用"和真的掏钱是两回事。

**周三**，竞品分析写了20页PPT。结论是"差异化竞争"。但你也知道，这四个字约等于什么都没说。

**周四**，产品方案列了30个功能。每个都有人说重要。你不知道该砍哪个。

**周五**，品牌那边要广告语。"极致体验，尊享品质"——说出来连你自己都不信，但你想不出别的。

**这些不是某一个人的困境，是产品工作中反复出现的问题。** 而这些问题之间是有逻辑顺序的：想不清楚目标，就看不懂用户；看不懂用户，就不知道做不做；不知道做不做，竞争分析就是自嗨；分析再好，落不了地也是零；最后好不容易做出来了，用户记不住你。

每一步断裂，后面全部白做。

---

## 解法：6 个基础 Skill + 1 个编排 Skill

我们把产品从 0 到 1 的全流程，拆成了 6 个独立的分析模块，每个模块解决一个明确的问题，背后有具体的方法论支撑——不是空泛的框架，而是能直接用的工具。

然后用一个编排型 Skill 把它们串起来，实现一键全流程诊断。

```
                    ┌─────────────────┐
                    │   ⑦ 产品圆桌    │  ← 编排层：串联全流程
                    └──────┬──────────┘
                           │
        ┌──────┬──────┬──────┬──────┬──────┐
        ①      ②      ③      ④      ⑤      ⑥
     定义需求 理解用户 做不做 竞争分析 执行策略 传播策略
```

可以单独调，也可以一键全跑。

---

## 每个 Skill 解决什么？

### ① 定义需求——"到底要解决什么问题？"

老板说"做一个平台"，你需要把这句话变成一个具体的、可验证的目标。

"我想做一个很牛的产品"不是目标，是幻想。"3个月让1000个用户每周用3次"才是目标。

**用到的工具**：
- **傅盛·封闭目标**：开放问题→封闭目标（谁/什么/多少/约束）
- **JTBD**：用户"雇用"你的产品完成什么任务？
- **梁宁·痛爽痒**：这个需求是恐惧驱动、即时满足、还是理想自我？
- **存在感公式**：`我是___，我通过___证明自己___`

---

### ② 理解用户——"用户是谁，他在想什么？"

用户调研做完了，但你还是不理解用户。因为你看到的是回答，不是动机。

最有价值的信息往往藏在用户**没做的事**和**凑合着用的土办法**里。

**用到的工具**：
- **五层画像模型**：从外在行为到存在感内核，像剥洋葱一样拆解
- **大明/笨笨/小闲**：用户的行为模式，决定产品策略
- **力场分析**：什么在推用户前进，什么在拉他不动
- **情绪刻度**：愉悦、爽、不爽、愤怒、恐惧——恐惧是最强的产品驱动力
- **贪嗔痴**：人性三毒，用户行为的底层操作系统
- **反面画像**：定义"谁不是你的用户"，比知道"是谁"更有用

---

### ③ 做不做——"值不值得投入？"

很多产品不是死在做不好，而是死在不该做。

这个 Skill 干的事很简单：算账。用户价值是正还是负？商业上能不能赚回来？

**用到的工具**：
- **马化腾·马氏三问**：入口快筛——你擅长吗？不做用户损失什么？能保持多大优势？
- **俞军·用户价值公式**：`(新体验 - 旧体验) - 替换成本`
- **五维切换成本**：认知/信任/迁移/金钱/时间，任一项太高用户就不换
- **曹政·流量变现公式**：`收入 = 流量 × 转化率 × 客单价`
- **信息不对称**：你有没有别人还不知道的东西？没有就是硬抗
- **ROIC vs WACC**：投资回报 > 资本成本才是在创造价值

最后给一个明确判定：**做、改一改、还是算了。**

---

### ④ 竞争分析——"凭什么是你赢？"

"差异化竞争"这四个字等于什么都没说。你需要的是：对手会怎么反击你？你有什么他进不来的壁垒？

**用到的工具**：
- **傅盛·破局点**：够不够边缘、锋利、能不能自增长
- **Greenwald·护城河三重测试**：供给优势、需求锁定、规模经济
- **博弈论**：囚徒困境 / 进入独占博弈，用逆向归纳法预测对手
- **特许经营区边界**：你的优势有边界，出了边界你跟谁都没区别
- **EPV**：用财务数字量化竞争优势到底值多少钱

---

### ⑤ 执行策略——"具体怎么落地？"

30个功能全想做，资源只够做3个。怎么选？

答案是：不选3个。选1个。

**用到的工具**：
- **傅盛·All in**：找到那个"1"，其他全砍或降级
- **张小龙·极简规则**：核心规则能不能一句话说清？需要说明书就已经失败了
- **周鸿祎·小白思维**：3秒看不懂？重做
- **微创新/单点突破**：1个功能做到120分 > 10个功能60分
- **Kathy Sierra·用户成就感**：不做更好的相机，让用户成为更好的摄影师
- **Savoia·预型测试**：假门测试、机械土耳其人、一人公司——花最少的钱验证最关键的假设
- **曹政·先人工再技术**：能用Excel跑通的事，别先写代码
- **看原始日志**：别看PV/UV，看用户在哪一步停下来了

---

### ⑥ 传播策略——"怎么让人记住你？"

"极致体验，尊享品质"——把品牌名遮住，换成任何竞品都能用。这种文案等于零。

好文案不是你说一句话给顾客听，而是你设计一句话让顾客说给别人听。

**用到的工具**：
- **华与华·播传理论**：文案的检验标准是"顾客愿不愿意替你传"
- **四大生成算法**：填空指令法、顺口溜法、具体承诺法、场景寄生法——并行出稿
- **双重质检**：邻居闲聊测试（说出来尴尬就扔）+ 100年测试（100年前的人听不懂就扔）
- **书面语黑名单**："极致"、"赋能"、"臻选"——检测到就强制替换成人话

---

### ⑦ 产品圆桌——"帮我从头到尾跑一遍"

这是编排型 Skill。你只需要说一句"我想做一个XX"，它自动串联①到⑥走完全流程。

它有几个基础 Skill 没有的能力：

| 能力 | 说明 |
|------|------|
| **上下文传递** | 每个阶段的结论自动注入下一阶段，前后连贯 |
| **分支决策** | "做不做"判定为"算了"→ 直接终止，不浪费后续分析 |
| **矛盾检测** | 用户画像说"怕麻烦"，但执行策略设计了复杂流程→ 自动报警 |
| **大师辩论** | 傅盛/俞军/张小龙/周鸿祎/曹政/Kathy Sierra 六个视角碰撞 |

---

## 怎么用？

**场景一**：只想验证一个想法靠不靠谱
→ 单独调用「③ 做不做」

**场景二**：想深入了解目标用户
→ 单独调用「② 理解用户」

**场景三**：产品做好了，需要一句广告语
→ 单独调用「⑥ 传播策略」

**场景四**：有个完整想法，想全面诊断
→ 调用「⑦ 产品圆桌」，自动走完全程

**场景五**：已经做完了定义需求，想继续
→ 从任意一步开始单独调用，按需组合

每个 Skill 都能独立运行，也能被产品圆桌串联调用。像乐高积木一样——单块有用，拼起来更强。

## 背后的方法论来源

这组 Skill 不是凭空编的。每一个工具都有明确的理论出处：

| 来源 | 贡献 |
|------|------|
| **傅盛** | 封闭目标、破局点、All in |
| **俞军** | 用户价值公式、产品即交易 |
| **张小龙** | 极简规则、产品演化、用完即走 |
| **周鸿祎** | 小白思维、微创新、单点突破 |
| **曹政** | 流量公式、信息差、先人工、看日志 |
| **马化腾** | 马氏三问 |
| **梁宁** | 痛点/爽点/痒点、情绪刻度 |
| **Kathy Sierra** | 用户成就感（Badass原则） |
| **Alberto Savoia** | 预型测试（Pretotyping） |
| **Bruce Greenwald** | 护城河三重测试、EPV |
| **华与华** | 播传理论、文化母体 |
| **JTBD理论** | 待办任务 |
| **博弈论** | 囚徒困境、逆向归纳 |

不是搬运理论，是把这些理论变成了一步步可执行的操作流程。

---

## 一句话总结

产品工作中最贵的浪费，不是做得慢，而是做了不该做的事。

这组 Skill 做的就是一件事：**在你投入资源之前，帮你把该想清楚的问题全部想清楚。**
