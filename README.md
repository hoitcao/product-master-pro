# 🧠 产品大师 Pro

**7 个 AI Skill，覆盖产品从 0 到 1 的全流程。**

做产品最难的事，不是写代码，而是想清楚。

---

## 这是什么？

产品大师 Pro 是一组结构化的 AI Prompt（Skill），把产品经理从"定义需求"到"传播策略"的全流程，拆成了 **6 个独立的分析模块 + 1 个编排模块**。每个模块背后有具体的方法论支撑——不是空泛的框架，而是能直接用的工具。

```
                    ┌─────────────────┐
                    │   ⑦ 产品圆桌    │  ← 编排层：串联全流程
                    └──────┬──────────┘
                           │
        ┌──────┬──────┬──────┬──────┬──────┐
        ①      ②      ③      ④      ⑤      ⑥
     定义需求 理解用户 做不做 竞争分析 执行策略 传播策略
```

可以单独调用任何一个 Skill，也可以用「产品圆桌」一键串联全流程。

---

## 为什么做这个？

产品经理的日常大概是这样的：

- **周一**，老板说"我想做一个XX平台"。你点头说好，但心里没底——这个"平台"到底要解决什么问题？
- **周二**，做了份用户调研。100个人说"挺好的，我会用"。但你知道，问卷上说"会用"和真的掏钱是两回事。
- **周三**，竞品分析写了20页PPT。结论是"差异化竞争"。但这四个字约等于什么都没说。
- **周四**，产品方案列了30个功能。每个都有人说重要。你不知道该砍哪个。
- **周五**，品牌那边要广告语。"极致体验，尊享品质"——说出来连你自己都不信。

**这些问题之间有逻辑顺序**：想不清楚目标，就看不懂用户；看不懂用户，就不知道做不做；不知道做不做，竞争分析就是自嗨；分析再好，落不了地也是零；最后好不容易做出来了，用户记不住你。

每一步断裂，后面全部白做。

## 7 个 Skill 一览

| # | Skill | 解决什么 | 核心方法论 |
|---|-------|---------|-----------|
| ① | **[定义需求](skill-1-定义需求.md)** | 把模糊想法变成具体目标 | 傅盛·封闭目标、JTBD、梁宁·痛爽痒、存在感公式 |
| ② | **[理解用户](skill-2-理解用户.md)** | 从行为挖出深层动机 | 五层画像、大明/笨笨/小闲、力场分析、贪嗔痴 |
| ③ | **[做不做](skill-3-做不做.md)** | 算账，值不值得投入 | 马氏三问、俞军·用户价值公式、流量变现公式、ROIC |
| ④ | **[竞争分析](skill-4-竞争分析.md)** | 护城河、对手、博弈 | 傅盛·破局点、Greenwald·护城河测试、博弈论、EPV |
| ⑤ | **[执行策略](skill-5-执行策略.md)** | 砍功能、出计划、先验证 | All in、极简规则、小白思维、预型测试 |
| ⑥ | **[传播策略](skill-6-传播策略.md)** | 造一句顾客替你传播的口号 | 华与华·播传理论、四大算法、双重质检 |
| ⑦ | **[产品圆桌](skill-7-产品圆桌.md)** | 一键全流程诊断 | 编排①→⑥ + 矛盾检测 + 大师辩论 |

## 快速开始

### 方式一：直接复制 Prompt 使用

1. 打开任意一个 `skill-*.md` 文件
2. 将其内容作为 System Prompt 粘贴到你的 AI 对话中
3. 输入触发词开始使用

**触发词示例**：

| Skill | 触发词 |
|-------|--------|
| 定义需求 | `定义需求`、`帮我想清楚`、`我想做一个XX` |
| 理解用户 | `理解用户`、`用户洞察`、`用户为什么会这样` |
| 做不做 | `做不做`、`值不值得做`、`帮我算笔账` |
| 竞争分析 | `竞争分析`、`护城河`、`市场能不能进` |
| 执行策略 | `执行策略`、`怎么落地`、`MVP怎么做` |
| 传播策略 | `传播策略`、`帮我写文案`、`品牌口号` |
| 产品圆桌 | `产品圆桌`、`完整诊断`、`帮我从头到尾看看` |

### 方式二：作为 AI Agent Skill 集成

将 7 个 `skill-*.md` 文件配置为你的 AI Agent 的 Skill，根据用户输入自动激活对应模块。

### 方式三：运行自动化测试

```bash
# 安装依赖
pip install -r requirements.txt

# 设置 API Key
export ANTHROPIC_API_KEY="your-key-here"

# 运行全部测试
python test_skills.py

# 只测试指定 Skill
python test_skills.py --skills 1 3 6

# 指定模型
python test_skills.py --model claude-sonnet-4-20250514
```

## 使用场景

| 场景 | 用哪个 Skill |
|------|-------------|
| 只想验证一个想法靠不靠谱 | ③ 做不做 |
| 想深入了解目标用户 | ② 理解用户 |
| 产品做好了，需要一句广告语 | ⑥ 传播策略 |
| 有个完整想法，想全面诊断 | ⑦ 产品圆桌 |
| 已做完定义需求，想继续下一步 | 从任意一步开始，按需组合 |

每个 Skill 都能独立运行，也能被产品圆桌串联调用。像乐高积木一样——单块有用，拼起来更强。

## 方法论来源

这组 Skill 不是凭空编的。每一个工具都有明确的理论出处：

| 来源 | 贡献 |
|------|------|
| **傅盛** | 封闭目标、破局点、All in |
| **俞军** | 用户价值公式、产品即交易 |
| **张小龙** | 极简规则、产品演化、用完即走 |
| **周鸿祎** | 小白思维、微创新、单点突破 |
| **曹政** | 流量公式、信息差、先人工、看日志 |
| **马化腾** | 马氏三问 |
| **梁宁** | 痛点/爽点/痒点、情绪刻度 |
| **Kathy Sierra** | 用户成就感（Badass 原则） |
| **Alberto Savoia** | 预型测试（Pretotyping） |
