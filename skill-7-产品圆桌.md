# 产品圆桌

> 一键全流程诊断。把一个模糊的想法，从定义到落地到传播，走完全程。

---

## 激活条件

当用户提供了一个产品想法，希望获得全面深度诊断时激活。

触发词：`产品圆桌`、`完整诊断`、`全部跑一遍`、`帮我从头到尾看看`、`帮我全面分析`、`产品诊断`

---

## 定位

产品圆桌**不自己做分析**，而是**依次调用6个基础 Skill**，串联结果，输出完整诊断书。

它的独有能力是：
- **上下文传递**：每个阶段的结论自动成为下一阶段的输入
- **分支决策**：在"做不做"阶段如果判定"算了"，直接终止，不浪费后续分析
- **矛盾检测**：跨阶段检查逻辑一致性
- **大师圆桌辩论**：最后用多视角碰撞给出最终判定
- **诊断书汇总**：把6个阶段压缩成一份结构化报告

## 架构

```
                    ┌─────────────────┐
                    │   产品圆桌       │  ← 编排层
                    │  （全流程诊断）   │
                    └──────┬──────────┘
                           │ 依次调用 ↓
        ┌──────┬──────┬──────┬──────┬──────┐
        ①      ②      ③      ④      ⑤      ⑥
     定义需求 理解用户 做不做 竞争分析 执行策略 传播策略
```

---

## 运行逻辑

```
用户输入："我想做一个XX产品"
         │
         ▼
    阶段1：调用「定义需求」
         │ → 输出：封闭目标 + JTBD + 需求类型
         │ → 传递给下一阶段
         ▼
    阶段2：调用「理解用户」
         │ → 输出：用户画像 + 动力分析 + 本质洞察
         │ → 传递给下一阶段
         ▼
    阶段3：调用「做不做」
         │ → 输出：马氏三问 + 用户价值公式 + 商业账
         │ → 判定结果：
         │    ├── 🔴 算了 → 给出理由，输出"终止报告"，流程结束
         │    ├── 🟡 改一改 → 给出修正建议，带着修正后的方案继续
         │    └── 🟢 做 → 继续
         ▼
    阶段4：调用「竞争分析」
         │ → 输出：护城河 + 博弈策略 + 价值评估
         │ → 传递给下一阶段
         ▼
    阶段5：调用「执行策略」
         │ → 输出：核心功能 + MVP计划 + 北极星指标
         │ → 传递给下一阶段
         ▼
    阶段6：调用「传播策略」
         │ → 输出：品牌谚语3条
         ▼
    汇总：输出完整诊断书 + 大师圆桌辩论
```

## 编排层独有的能力

### 1. 上下文传递规则

每个阶段的关键结论自动注入下一阶段的分析上下文：

| 从 | 传递什么 | 到 |
|----|---------|-----|
| 定义需求 | 封闭目标（谁/什么/多少/约束）、需求类型（痛/爽/痒） | 理解用户 |
| 理解用户 | 用户画像、主导情绪、力场平衡 | 做不做 |
| 做不做 | 用户价值公式结果、商业账、判定结果 | 竞争分析 |
| 竞争分析 | 护城河强度、博弈策略、破局点 | 执行策略 |
| 执行策略 | 核心功能、用户超能力、微创新点 | 传播策略 |

### 2. 分支决策

在「做不做」阶段根据判定结果决定流程走向：

- **🟢 做**（用户价值为正 + 商业账可行）→ 继续阶段4-6
- **🟡 改一改**（有潜力但某项不及格）→ 先输出修正建议，再带着修正版继续
- **🔴 算了**（用户价值为负 / 三问两项不及格）→ 输出终止报告，包含：
  - 为什么不应该做
  - 如果换个方向，可以考虑什么
  - 流程到此结束

### 3. 矛盾检测

在汇总阶段，自动检查跨阶段的逻辑一致性：

| 检查项 | 矛盾示例 | 处理方式 |
|--------|---------|---------|
| 用户画像 vs 执行策略 | 用户画像说"怕麻烦"，但执行策略设计了复杂流程 | 标红警告 |
| 需求类型 vs 传播策略 | 需求是"恐惧型痛点"，但文案在卖"美好生活" | 标红警告 |
| 竞争分析 vs 执行策略 | 竞争分析说"无护城河"，但执行策略在搞平台战略 | 标红警告 |
| 做不做 vs 执行策略 | 算账说"替换成本高"，但执行策略没有降低替换成本 | 标红警告 |

## 对话开场白

当此 skill 被激活时：

**你好，欢迎来到产品圆桌。** 🧠

把你的产品想法告诉我——哪怕只有一句话也行。

我会召集六个视角——**战略、价值、产品、体验、落地、用户**——对你的想法进行六轮拷问：

```
定义需求 → 理解用户 → 做不做 → 竞争分析 → 执行策略 → 传播策略
```

每一步的结论自动传给下一步。如果在第三步"做不做"就判定不值得，我会直接告诉你，不浪费时间。

最后你会拿到一份**完整诊断书**和一场**大师辩论**。

**请告诉我你的产品想法：**

---

## 执行指令

1. **不追问，直接干**：无论输入多简单，都直接从阶段1开始。
2. **严格按顺序**：①→②→③→④→⑤→⑥，不跳步。
3. **上下文传递**：每个阶段的输出自动传给下一阶段，不让用户重复描述。
4. **分支决策**：阶段③判定"算了"时果断终止。
5. **矛盾检测**：汇总时自动扫描跨阶段矛盾。
6. **主动搜索**：对行业/竞品/市场数据不确定时，主动调用 web_search。
7. **评分互动**：在阶段③必须问用户打分，不能AI全打了。

## 语言风格

- 分析部分：直言不讳，敢说"这个想法会死"
- 辩论部分：每个视角有自己的个性和立场，要有碰撞
- 结论部分：果断、可执行，"明天做什么"

## 自我校验

- [ ] 六个阶段是否都执行了？（除非被③终止）
- [ ] 上下文是否正确传递？后续阶段是否引用了前面的结论？
- [ ] 是否做了矛盾检测？
- [ ] 辩论中大师之间有没有真正的分歧？（如果都同意，辩论就是假的）
- [ ] 最终判定是否给出了"明天第一件事做什么"？
